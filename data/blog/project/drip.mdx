## Expo CLI를 선택한 이유

React-Native CLI를 사용하여 시작하려고 했으나 Expo CLI의 장단점을 명확히 알고 넘어가고 싶어 일부로 Expo CLI를 선택하게 되었다.

단점이 명확하게 있지만 직접 체감해보고 싶었다. 그리고 앱 개발을 빨리 시작해보고 싶기도 했기 때문이다.

## Expo CLI 설치

```bash
npx create-expo-app my-app --template tabs
```

### npm 환경에서 yarn 환경 사용하고 싶을 때

**Expo**는 Metro Bundler가 node_modules 구조를 직접 참조하기 때문에 Yarn 4의 기본 PnP
모드보다 전통적인 node_modules 방식이 더 안정적이라고 한다.

```json
// .yarnrc.yml
nodeLinker: node-modules
```

### Expo Go를 통해 시작

Expo Go 앱을 설치하여 QR 코드를 스캔하여 바로 앱 개발을 시작할 수 있었다.

## 디자인 시스템 구축하기

**React-Native**에서 어떠한 UI 라이브러리를 사용 할 지 결정해야 했다.

레딧에서 **@shopify/restyle**를 사용하여 디자인 시스템을 구축할 때 개발자경험(DX)이 좋았다고 평이 좋았어서 이걸 이용하기로 했다.

나중에 익숙해지면 React Native CLI에서 **Unistyles** 혹은 **Tamagui** 로 진행 해봐야지.
